# 智能合规审核系统 - 前端开发计划

## 设计理念

**极简（集点）风格**：
- 清晰的视觉层次
- 最少的交互步骤
- 专注核心功能
- 高效的信息展示

参考：danduola.ai 的现代化设计风格

## 技术栈

- **框架**: Next.js 14 (App Router)
- **样式**: Tailwind CSS
- **UI组件**: shadcn/ui（基于Radix UI）
- **状态管理**: React Hooks
- **API请求**: fetch API
- **图标**: Lucide React

## 开发步骤

### 第一阶段：基础搭建（30分钟）
1. 初始化Next.js项目
2. 配置Tailwind CSS
3. 安装shadcn/ui组件
4. 创建基础布局（侧边栏、头部）
5. API调用封装

### 第二阶段：核心功能（45分钟）
6. 首页 - 条款匹配查询界面
7. 结果展示组件
8. 关键词搜索页面
9. 法规浏览页面

### 第三阶段：管理功能（30分钟）
10. 角色管理页面
11. 单据类型管理页面
12. 审核规则管理页面

### 第四阶段：配置功能（20分钟）
13. 设置页面 - AI模型配置
14. API Key管理
15. 匹配策略选择

### 第五阶段：优化完善（15分钟）
16. 响应式适配
17. 加载状态处理
18. 错误提示优化
19. 最终测试

**总计：约2.5小时**

## 特色功能

1. **一键查询**：最简化的交互流程
2. **实时搜索**：输入即搜索
3. **API配置向导**：引导式配置AI模型
4. **暗色模式**：支持主题切换
5. **响应式设计**：适配桌面和移动端
